🎯 Learning Outcomes

* Gained a clear understanding of the concept of **unit testing** and its importance in ensuring software quality and reliability.
* Learned how to design and test backend APIs using **Python and Pytest**, including writing test cases for different scenarios.
* Developed the ability to test frontend functionalities such as forms and user interactions using **Vitest and JSDOM**.
* Understood how to execute tests using the **Command Line Interface (CLI)** for both frontend and backend environments.
* Acquired knowledge of structuring test files and organizing project directories for better maintainability.
* Learned how to debug errors in both backend and frontend test cases and improve code quality through testing.
* Gained practical experience with **Git and GitHub** for version control, including staging, committing, and pushing code.
* Understood how to set up and configure **GitHub Actions** to automate testing workflows.
* Learned the integration of **CI/CD pipelines** to automatically run tests on every code push.
* Developed problem-solving skills by identifying and fixing issues related to dependencies, permissions, and environment configurations.
* Gained exposure to real-world development practices where automated testing is essential before deployment.
* Improved understanding of cross-platform compatibility issues (Windows vs Linux environments) in testing.
* Learned how to manage project dependencies using **pip (Python)** and **npm (Node.js)**.
* Understood the importance of maintaining clean repositories using `.gitignore` and avoiding unnecessary files.
* Enhanced overall confidence in developing, testing, and deploying full-stack applications with proper validation mechanisms.
* Built a strong foundation for applying testing strategies in future software development projects and industry-level applications.
